    # 1.插槽

    插槽（slot）指组件封装期间，为用户预留的内容的占用符 基本用法

    //mycom组件
//<slot>默认内容</slot> 可以在插槽中预留内容 
<div>
  <slot></slot> 
</div>

    # 具名插槽

    如果在封装组件是需要预留多个插槽结点， 则需要为每个插槽指定具体的name名称。

    没有指定name名称的插槽，会有隐含的名称'default'

    //mycom组件
<div>
  <slot name='header'></slot> 
</div>

    # 作用域插槽

    作用域插槽指：为预留的slot插槽绑定props数据

    //mycom组件
<div>
  <slot name='header' :msg="message"></slot> 
</div>

    # 2.自定义指令

    # 私有自定义指令

    在每个vue组件中，可以在directive节点下声明私有自定义指令

    <input v-focus/>
-----
directives: {
  focus:{
    //当被绑定的元素插入到DOM中，自动触发mounted函数
    mounted(el){
      el.focus() //让被绑定的元素自动获得焦点
    }
  }
}

    # 全局自定义指令

    全局共享的自定义指令需要通过"单页面应用程序的实例对象"进行声明

    //main.js
app.directive('focus',{
  mounted(el){
    el.focus()
  }
})

    # updated函数

    updated函数会在每次DOM更新完成后被调用。

    mounted函数只在元素第一次插入DOM时被调用

    app.directive('focus',{
  mounted(el){
    el.focus()
  }
  updated(el){
    el.focus()
  }
})

//注意： 如果mounted和updated函数中逻辑完全相同可简写；
app.directive('focus',(el)=>{
    el.focus()
})

    # 指令的参数值

    再绑定指令时，可以通过"="的形式为指令绑定具体的参数值

    <p v-color="'red'"></p>

//自定义指令
app.directive('color',(el,binding)=>{
  el.style.color = binding.value
})
